# 文件

`文件` 是一个 IntelliJ IDEA 插件，用于在 IDE 侧边栏浏览项目外部的本地文件夹。

它适合需要同时查看多个代码目录的场景：你可以把 Java 后端、前端项目、C++ 模块、脚本目录或文档目录一起加入浏览，但这些目录不会被加入当前 IDEA Project Model。

## 主要用途

- 添加项目外的本地文件夹，并在独立工具窗口中预览目录结构。
- 双击文件后使用 IDEA editor 打开文件。
- 同时浏览 Java、前端、C++ 等多种类型项目文件。
- 在 vibe coding 场景中，把多个相关项目目录放在同一个 IDE 视野里查看。
- 避免为了临时查看文件而把外部目录加入当前项目。

## 使用方式

1. 安装插件后，打开 IDEA 侧边栏里的 `文件` 工具窗口。
2. 点击工具栏的 `Add Folder` 按钮，选择一个项目外部的本地目录。
3. 在树中展开目录并预览文件。
4. 双击文件可直接在 IDEA editor 中打开。
5. 可继续添加多个目录，用于同时查看后端、前端、C++、配置或文档项目。

## 不会做的事情

插件只负责浏览和打开外部文件夹，不会：

- 添加 `Content Root`。
- 修改 `.idea/modules.xml`。
- 触发 Maven 或 Gradle import。
- 把外部目录纳入当前项目索引或重构范围。

## 适用场景

- 一个业务同时包含 Java 服务、前端工程、C++ 模块，需要放在同一个 IDE 里快速查看。
- 当前 IDEA project 只打开主仓库，但需要参考旁边的脚本、配置、SDK 或文档目录。
- vibe coding 时需要让多个项目文件同时可见，方便快速切换上下文。
- 临时浏览外部目录，但不希望改变当前项目结构。
